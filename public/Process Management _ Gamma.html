Process Management
Today, we will explore the fundamentals of process management, including process definition, the functions of processes, and inter-process communication.

What is a Process?

1

An Instance of a Program
It is an instance of a program in execution, representing an asynchronous activity.

2

Animated Spirit
It is often referred to as the "animated spirit" of a procedure.

3

Dispatchable Unit
A process is an individually schedulable unit by an operating system.
Process States

1

Running State
The process currently executing on the CPU is in the Running state.

2

Blocked State
A process waiting for something (e.g. I/O) to complete is in the Blocked state.

3

Ready State
Many processes may be ready and waiting, but only one process can run on any processor at any instant.
A Seven Step Process Model
1
Blocked - Blocked Suspend
When all processes are blocked, the OS may remove a blocked process to bring an unblocked process into memory.
2
Blocked Suspend - Ready Suspend
Occurs when the event for which the process has been waiting occurs.
3
Ready - Ready Suspend
When there are no blocked processes and memory needs to be freed up for performance reasons.
Process Control Block
Information Contained
A Process Control Block (PCB) contains essential information about a specific process or program running on a computer system.
Contents
It includes process state, program counter, CPU registers, CPU scheduling information, memory-management information, accounting information, and I/O status information.
